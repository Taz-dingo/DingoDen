# Kana Status

## 当前阶段

项目目前处于“早期原型可运行”阶段。

已具备基础浏览能力，但离真正可用的学习产品还有明显差距。

## 已有能力

- 已有 Astro + React + Tailwind 的基础项目结构
- 已有单页面入口
- 已支持平假名 / 片假名切换
- 已支持点击假名查看详情
- 已在详情里展示罗马音、类型、部分字源
- 当前构建可通过

## 当前缺口

### 数据层

- 五十音表数据结构过于僵硬，强制每行固定五列
- `や行`、`わ行` 用重复字符填补空位，表达不准确
- 当前模型不利于未来扩展浊音、拗音与特殊音节

### 交互层

- 详情弹窗仍较基础，缺少更强的学习导向
- 没有发音能力
- 没有练习模式
- 没有进度或反馈机制

### 视觉层

- 当前页面更像技术演示，不像学习产品
- 层级比较单薄，缺少明确的总览 / 详情节奏
- 暗色样式存在，但没有完整主题策略

### 可访问性

- 详情层尚未完整处理焦点、Esc、键盘可达性
- 发音与状态反馈机制尚不存在

## 最近确认的新方向

- 参考页的“总览 + 点击详情 + 字源 + 发音”结构很接近目标
- `kana` 后续会明确加入发音能力
- 从现在开始把文档沉淀视为正式工作流的一部分

## 当前判断

这个项目适合继续做，而且值得做。

原因不是它已经完成很多，而是它的目标边界足够清晰：

- 场景明确
- 内容聚焦
- 交互闭环天然成立
- 小而完整，适合逐步打磨

目前最应该做的不是继续堆小功能，而是先把基础模型、参考方向和实现顺序稳定下来。
