# Blog Docs

这套文档是当前 blog 单仓库的长期工作台，用来沉淀项目目标、设计哲学、当前状态和下一步任务。

## 组织方式

- `spec.md`
  - 回答“这个项目为什么存在、要做成什么样、哪些事情不做”
  - 适合记录设计哲学、产品定位、体验原则、非目标
- `plan.md`
  - 回答“接下来一段时间准备怎么推进”
  - 适合记录阶段目标、优先级、推进顺序
- `status.md`
  - 回答“项目现在到哪一步了”
  - 适合记录已完成、进行中、已知问题、最近变化
- `tasks.md`
  - 回答“现在具体要做什么”
  - 适合记录可执行任务、优先级、负责人/状态约定

## 维护约定

- `spec.md` 低频更新，只在方向、定位、设计原则发生变化时调整
- `plan.md` 中频更新，在阶段目标变化时调整
- `status.md` 高频更新，在完成一轮明显工作后刷新
- `tasks.md` 持续更新，保持可执行、可勾选、可排序

## 当前原则

- 文档边界只覆盖当前 blog 项目本身
- 优先记录对设计、内容、实现有直接指导价值的信息
- 保持短文档、强约束、少空话，避免写成“愿景口号墙”
