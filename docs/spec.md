# Blog Spec

## 项目定位

`Dingo's Den` 是一个偏 AI / agents / applied AI 的个人博客。

它不是内容农场，也不是“每周必须更新”的媒体站点；它更像一个经过设计打磨的个人笔记空间，用来沉淀：

- agent workflow 与 AI 工程中的坑与经验
- prompt、上下文、工具调用与编排相关的理解
- AI 产品体验与工程交界处的细节判断
- 值得反复引用的实践型文章

## 核心目标

1. 做成一个有明确气质的个人博客，而不是模板感强的技术站或 AI 资讯流
2. 让阅读体验比“信息堆叠”更重要：清楚、安静、可停留
3. 让内容结构支持长期积累，而不是只适配少量文章
4. 让样式系统足够稳定，后续新增文章不需要频繁推倒重来

## 设计哲学

### 1. Calm, editorial, deliberate

界面应该像一本整理过的个人刊物，而不是一个组件展示页。

关键词：

- 安静
- 克制
- 有留白
- 强层级
- 少而准的强调

### 2. Composition before components

先看整体构图、版面、节奏、字体、留白，再考虑组件样式。

这意味着：

- 首屏要有明确视觉中心
- 页面分区要一眼能分辨主次
- 不依赖大量卡片、徽标、阴影制造“设计感”

### 3. Content is the hero

博客的主角是文章本身，不是装饰性 UI，也不是追热点的信息流。

因此：

- 标题和摘要层级必须稳
- 文章正文必须可长时间阅读
- 目录、日期、分类、归档都应该服务于阅读与检索

### 4. One system, not many vibes

页面之间可以有变化，但不能像不同模板拼起来。

需要统一的：

- 字体系统
- 间距系统
- 标题层级
- 颜色语气
- 链接与交互反馈

## 体验原则

### 首页

首页不是简单文章列表，而是站点气质的第一页。

首页应承担：

- 清楚表达站点是谁、写什么
- 给出一个有记忆点的第一视觉
- 让用户自然进入最新文章或归档

### 文章页

文章页是最重要的页面。

文章页应优先保证：

- 标题、摘要、日期信息有明确层级
- 正文宽度、行高、段落距离适合长时间阅读
- 代码块、注释、提示块有稳定样式
- TOC 是辅助，不抢主内容

### 归档页

归档页应该像索引，而不是另一个首页。

它的目标是：

- 快速扫描
- 低干扰
- 方便回找

## 内容原则

- 文章优先写“可复用经验”，少写纯情绪短帖
- 摘要要解释文章价值，不只是重复标题
- 标题应该明确，不靠夸张措辞吸引点击
- 示例代码、流程图和交互 demo 应该服务理解，而不是装饰

## 非目标

当前阶段不追求：

- 复杂 CMS
- 多作者体系
- 评论系统
- 花哨动效站
- 重型后台与运营工具
- 大而全的标签/筛选系统

## 技术边界

当前 `blog` 项目基于：

- Astro
- Tailwind CSS
- MDX
- React / Vue（用于少量交互示例）

技术上的目标不是“堆更多库”，而是：

- 保持静态站的轻量与稳定
- 在必要处引入交互或实验性演示
- 让内容与设计系统长期可维护
