# Blog Plan

## 当前阶段主题

当前阶段主题是：**把 blog 从“能用的旧项目”推进到“有明确审美和维护节奏的长期项目”。**

这意味着我们不是先拼功能，而是先把基础盘稳住：

- 视觉语言
- 字体系统
- 页面层级
- 文档与工作方式
- 后续改造的优先级

## 阶段划分

### Phase 1 — Foundation

目标：建立统一视觉系统与项目文档。

包含：

- 首页 / 文章页 / 归档页基础改版
- 全局样式 token 收束
- 字体系统统一
- 建立 `spec / plan / status / tasks`

### Phase 2 — Reading Experience

目标：把阅读体验打磨到稳定可复用。

包含：

- 代码块主题与排版优化
- 提示块、引用块、表格等 MDX 内容样式统一
- 移动端阅读体验补齐
- 图片、插图、hero 图策略建立

### Phase 3 — Content System

目标：让内容增长时结构不崩。

包含：

- 分类/路径命名规则
- 封面图与摘要规范
- 归档与列表页扫描体验优化
- SEO / metadata / social card 完善
- 评估用 MCP 接入既有 Markdown 笔记导入与发布工作流

### Phase 4 — Signature Details

目标：补足品牌感与完成度。

包含：

- 更成熟的代码高亮方案
- 局部动效优化
- favicon / OG image / sharing assets
- 必要的微交互精修

## 近期优先顺序

1. 固化视觉和字体系统
2. 明确文档和维护规则
3. 优化文章阅读体验
4. 再考虑内容规模化与传播细节

## 推进原则

- 一次只做少量高价值改动
- 每轮改动尽量能直接在页面上看到收益
- 不为了“现代感”引入无必要依赖
- 所有新增设计都要回答：是否提升阅读和品牌识别
